# 5장 개념적 설계 기초

## 0. 요구 분석 내용

1. 게임 회사는 여러 개의 모바일 게임을 제공하고 있다.
2. 회원 가입을 하려면 id와 pw를 설정해야하며, 이름과 닉네임, 휴대폰 번호가 필요하다.
3. 게임 정보는 게임 번호와 게임명, 현재 버전, 최초 출시일, 현재 버전 출시일을 관리한다.
4. 회원이 각 게임을 언제 시작해서 언제 종료했으며, 몇 점을 획득했는지 이력을 기록한다.

## 9. ER 다이어그램 작성 전 체크리스트

### 개체

| No. | 내용 | 확인 |
| --- | --- | --- |
| 1 | 실제 데이터가 1개만 존재하는 개체가 식별되었는가? | o (회사 제외) |
| 2 | 속성이 1개인 개체가 식별되었는가? | x |
| 3 | 개체 이름이 중복되었는가? | x |
| 4 | 각 개체의 기본키가 결정되었는가? | o |
| 5 | 각 개체를 설명하는 일반 속성이 모두 식별되었는가? | o |
| 6 | 개체 가운데 관계로 표현하면 더 적합한 것이 있는가? | x |

### 관계

| No. | 내용 | 확인 |
| --- | --- | --- |
| 1 | 관계를 설명하는 설명자(일반 속성)가 모두 식별되었는가? | o |
| 2 | 관계 이름이 동사로 표현되었는가? | o |
| 3 | 관계 이름이 현재 시제로 표현되었는가? | o |
| 4 | 관계 이름이 개체 간의 관련성을 명확히 표현하고 있는가? | o |
| 5 | 관계 이름이 중복되었는가? | x |
| 6 | 관계 유형을 정확히 결정하였는가? | o |

### 속성

| No. | 내용 | 확인 |
| --- | --- | --- |
| 1 | 속성 이름에 개체나 관계 이름이 포함되었는가? | x |
| 2 | 속성 이름으로 의미가 불분명한 약어가 사용되었는가? | x |
| 3 | 속성의 성격을 명확히 인식하기에 쉬운 이름이 부여되었는가? | o |
| 4 | 속성 이름을 부여하는 데 일정한 규칙이 있는가? | o |
