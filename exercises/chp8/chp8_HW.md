## 3

회원이 어떤 고등학교를 졸업했는지 표현한 ER 다이어그램의 변환

고등학교(고등학교번호 PK, 고등학교명, ...)
회원(회원번호 PK, 회원명, ..., 고등학교번호 FK)

관계 릴레이션 단순화 표현:

회원(회원번호 PK, 회원명, ..., 고등학교번호 FK)
고등학교(고등학교번호 PK, 고등학교명, ...)

## 4

지역별로 회원이 어떤 카페에 가입했는지 표현한 ER 다이어그램의 변환

지역(지역번호 PK, 지역명, ...)
회원(회원번호 PK, 회원명, ...)
카페(카페번호 PK, 카페명, ...)

가입하다(
  지역번호 FK,
  회원번호 FK,
  카페번호 FK,
  PRIMARY KEY(지역번호, 회원번호, 카페번호)
)

## 5

학생이 어떤 외부 활동에 참여했는지 표현한 ER 다이어그램의 변환

학생(학번 PK, 이름, ...)
외부활동(활동번호 PK, 활동명, ...)

참여하다(
  학번 FK,
  활동번호 FK,
  PRIMARY KEY(학번, 활동번호)
)

## 6

모든 학생이 반드시 하나의 방과후 활동에 참가하고, 학생 1명이 각 방과후 활동의 대표를 맡는 경우

방과후활동(
  활동번호 PK,
  활동명,
  ...,
  대표학생학번 FK UNIQUE NOT NULL
)

학생(
  학번 PK,
  이름,
  ...,
  활동번호 FK NOT NULL
)

단순화 핵심:

학생 릴레이션에 활동번호 FK를 포함한다.
방과후활동 릴레이션에 대표학생학번 FK를 포함한다.

## 7

교수가 어떤 연구회에 가입했는지 표현한 ER 다이어그램의 바람직한 변환
단, 모든 교수가 연구회에 가입하는 것은 아님

연구회(연구회번호 PK, 연구회명, ...)
교수(교수번호 PK, 교수명, ..., 연구회번호 FK NULL)

## 8

학생이 다른 학생을 멘토링하는 순환 관계
단, 모든 학생은 멘티이고 일부 학생만 멘토 역할을 함

학생(
  학번 PK,
  이름,
  ...,
  멘토학번 FK NOT NULL
)

추가 제약조건:

멘토학번은 학생(학번)을 참조한다.
한 멘토가 담당할 수 있는 멘티 수는 최대 5명이다.

## 9

콘서트 개최 시 뮤지션, 기획사, 콘서트홀이 연관되는 `콘서트개최하다` 관계 릴레이션

콘서트개최하다(
  뮤지션번호 FK,
  기획사번호 FK,
  콘서트홀번호 FK,
  PRIMARY KEY(뮤지션번호, 기획사번호, 콘서트홀번호)
)

관계에 속성이 있다면 다음처럼 포함한다.

콘서트개최하다(
  뮤지션번호 FK,
  기획사번호 FK,
  콘서트홀번호 FK,
  개최일자,
  공연시간,
  ...,
  PRIMARY KEY(뮤지션번호, 기획사번호, 콘서트홀번호)
)

  응시자번호 FK,
  자격증번호 FK,
  PRIMARY KEY(응시자번호, 자격증번호)
