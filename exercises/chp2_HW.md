5. 구분 수행 이유

 - 각 단계의 목적과 독자가 다름
 - 각 단계를 분리해 복잡한 설계 과정을 체계적으로 관리 가능하며, 단계별로 오류 검토, 수정하기에 용이함

6. 개념적 구조를 DBMS가 직접 이해할 수 없는 이유

 - 개념적 데이터 모델은 DB 설계자의 이해를 돕기 위한 추상적 개념으로 표현되어 있음
 - DBMS는 논리적 구조를 기반으로 이해를 하기에, 추상적 개념으로 이루어진 개념 구조를 이해 불가능함

7. 데이터 모델을 구성하는 세가지 요소

 - 데이터 구조: DB에 저장할 대상을 표현
 - 연산: 데이터 구조에서 허용되는 연산
 - 계약: 데이터 구조와 연산에 대한 논리적 제약 조건

8.개념적 데이터 모델의 구성 요소

 - 개체 타입: 속성들로 구성된 개체의 유형
 - 속성: 개체 타입을 구성하는 특성
 - 관계: 개체 타입들 사이의 연관성
